В Spring Boot помимо аннотации Value, можно использовать аннотацию ConfigurationProperies, с помощью которой можно сделать маппинг проперти файла -> Java POJO

Плюсы в отличие от Value:
- Объединение всех полей в одном классе (нет необходимости над каждым параметром писать Value)
- Можно использовать несколько пространств имен
- Проще использовать в сочетании со сложными структурами данных (List, Map, etc.)

Также круто то, что Spring Boot может автоматически обнаружить конфигурационные файлы, а также с помощью одного названия включать или исключать их
Для этого нужно чтобы название файла было типа application-....properties/yml

То есть можно использовать
application.yml в качестве основной конфигурации
а application-dev/application-prod для профайлов

В случае, если хочется самому управлять использованием тех или иных конфигов, это можно делать с помощью свойства spring.config.name и/или spring.config.location
