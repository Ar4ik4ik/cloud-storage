Суть АОП заключается в том, чтобы отделить бизнес логику и программные компоненты. То есть выделить "Аспекты" модульно и переиспользовать их.
	Логирование
	Транзакции
	Обработка исключений

Основные понятия:

Join Point - место в коде где применяется аспект
Advice - конкретное действие выполняемое на точке среза
Pointcut - определяет места где будет применятся advice

По сути что то типо декораторов в Python, но более гибкое и масштабируемое

Типы advice
- Before
- After returning (после успешного завершения)
- After throwing (если выброшено исключение)
- After
- Around (до и после выполнения)

Два основных фреймворка:

AspectJ
	Используется в сложных сценариях, более трудный, имеет возможность управления на этапе компиляции
Spring AOP
	Встроен в Spring
	Более простой в применении
	Управление на уровне времени выполнения

Что можно писать в Pointcut:
- execution
	По сути указание методов с определенной сигнатурой
- within
	Указание применения к определенным типам и классам
- this
	Указание конкретного бина, нужно указывать ссылку на бин, в таком случае есть возможность получить экземпляр этого бина
- target
	Указание типа и применения для экземпляров этого типа, например разные реализации одного интерфейса
- args
	Сужает применение к случаям когда аргументы являются экземплярами указанных типов
- bean
	Указание конкретного бина
- @target
		Применение к классам, у которых аннотация указанного типа
- @args
		Применение к аргументам, у которых аннотации указанных типов
- @within
		Применение к методам внутри типов имеющих указанную аннотацию
- @annotation
		Применение к методам, у которых субъект (метод выполняемый в Spring AOP) имеет указанную аннотацию
